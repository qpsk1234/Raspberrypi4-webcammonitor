# 要件定義：Raspberry Pi 4 人間検知監視カメラアプリ

## 1. 目的
Raspberry Pi 4上で動作し、リアルタイムで人間を検知できる監視カメラアプリケーションを開発する。

## 2. 要件

### 2.1 機能要件
- **リアルタイムカメラフィード**: カメラ（USBまたはCSI接続）からの映像ソースを取得し処理する。
- **人間検知**: AI/機械学習モデルを使用して、フレーム内の人間を識別する。
- **アラート/通知**: 人間を検知した際に通知を送信、またはクリップを保存する（詳細は設計フェーズで決定）。通知はTelegramに送る。

- **録画**: 検知時に動画または画像を保存する。
- **ストリーム再生**: カメラの画像は常時モニター出力できるようにする。また、Webブラウザでもストリーム再生できるようにする。

### 2.2 非機能要件
- **パフォーマンス**: Raspberry Pi 4上で実用的なフレームレート（FPS）で処理可能であること。
- **低遅延**: 検知は最小限の遅延で行われること。
- **堅牢性**: 継続的に動作し、カメラの切断などにも適切に対応すること。

## 3. ユーザーストーリー
- ユーザーとして、侵入者が検知されたときに通知を受け取りたい。
- ユーザーとして、ストレージを節約するために人がいるときだけ録画したい。

## 4. スコープ
- 対象プラットフォーム: Raspberry Pi 4
- 言語: Python（Pi上のAI/ML処理に最適）
- 主要フォーカス: 人間検知の精度と効率
